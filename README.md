# Energy Economics Lab — Data Pipeline

This repo contains two independent data pipelines used for research on semiconductor/tech patent filers and US electricity prices.

---

## Repository Structure

```
Energy_Econ_Lab/
│
├── Company R&D/
│   ├── write_notebook.py          # Master script — run this to regenerate the notebook
│   ├── table_generation.ipynb     # Generated notebook — do not edit directly
│   ├── clean_companies.py         # Step 1: cleans and deduplicates the raw patent CSV
│   ├── edgar_cache.json           # Auto-generated cache of SEC EDGAR API responses
│   ├── sec_filers.csv             # Output: companies matched to a SEC CIK
│   ├── non_sec_filers.csv         # Output: companies with no SEC filing found
│   └── sec_financials.csv         # Output: final quarterly financial data table

└── Electricty Price Cleaning/

    ├── Average_retail_price_of_electricity.csv   # Raw EIA input file
    └── electricity_prices_tidy.csv               # Output: tidy long-format price data
```

## Pipeline 1 — Company R&D Financials

Fetches quarterly **R&D Expenses**, **Revenue**, and **Cost of Revenue** for semiconductor and tech patent filers from the SEC EDGAR XBRL API (and Financial Modeling Prep / Yahoo Finance for foreign companies).

### Step 1 — Clean the company list

```bash
python3 "Company R&D/clean_companies.py"
```

**Input:** `~/Downloads/companies_manually_cleaned.csv` (raw patent applicant data)  
**Output:** `~/Downloads/companies_cleaned.csv`

What it does:
- Collapses subsidiary/variant names into a single canonical parent company name (e.g. "TSMC Nanjing" → "Taiwan Semiconductor Manufacturing Company Limited")
- Deduplicates and assigns an `entity_type` (Parent / Subsidiary / Duplicate)
- Produces one row per canonical company with aggregated patent counts and citations

### Step 2 — Run the financial data notebook

Open `Company R&D/table_generation.ipynb` in VS Code (or Jupyter) and run all cells top to bottom.

**Important:** never edit `table_generation.ipynb` directly. It is fully generated by `write_notebook.py`. If you need to change pipeline logic, edit `write_notebook.py` and run:

```bash
python3 "Company R&D/write_notebook.py"
```

Then open / reload the notebook in VS Code.

#### Notebook cells overview

| Cell | What it does |
|------|-------------|
| 1 — Imports | Loads libraries, sets SEC User-Agent header and rate-limit delay |
| 2 — Known CIKs | Hardcoded CIK overrides for ~90 companies (verified against EDGAR) |
| 3 — Load companies | Reads `companies_cleaned.csv` |
| 4 — EDGAR tickers | Downloads the full SEC company ticker index (~1 MB, one call) |
| 5 — Match CIKs | Matches each company to a CIK via hardcoded list → exact name → partial name |
| 6 — Save split | Writes `sec_filers.csv` and `non_sec_filers.csv` |
| 7 — XBRL helpers | Defines tag lists and fetch functions for EDGAR XBRL API |
| 8 — Smoke test | Verifies Nvidia quarterly data against known values |
| 9 — Fetch all | Fetches XBRL data for all SEC filers (with caching — see below) |
| 10 — FMP / foreign | Fetches up to 40 quarters per foreign company from Financial Modeling Prep |
| 11 — EDGAR fallback | Retries failed EDGAR companies via Yahoo Finance |
| 12 — Build DataFrame | Combines all sources, adds `Foreign` and `Data Source` columns, drops pre-revenue companies |
| 13 — Coverage summary | Prints per-company filing counts and date ranges |
| 14 — Export | Writes `sec_financials.csv` |

### Data sources

| Source | Companies | Notes |
|--------|-----------|-------|
| SEC EDGAR XBRL | ~80 US domestic + some foreign filers | Free, no key needed. Supports both `us-gaap` and `ifrs-full` namespaces |
| Financial Modeling Prep (FMP) | 6 foreign companies (Sony, ASML, SAP, UMC, ASE, Tower Semi) | Up to 40 quarters of history. Requires API key (see below) |
| Yahoo Finance (yfinance) | Fallback for any company that fails EDGAR | ~4–5 quarters only; used as last resort |

### API keys

**FMP key** — set in cell 10 of the notebook:
```python
FMP_API_KEY = 'your_key_here'
```
Free keys are available at [financialmodelingprep.com](https://financialmodelingprep.com). Free tier allows 250 calls/day.

### EDGAR cache

Cell 9 saves all EDGAR responses to `Company R&D/edgar_cache.json`. On subsequent runs, companies already in the cache are loaded from disk instantly — no network call. This makes re-runs take seconds instead of minutes.

To force a full re-fetch (e.g. after adding new companies), set this flag at the top of cell 9:
```python
FORCE_REFRESH = True
```

### Foreign currency conversion

Foreign companies that report in non-USD currencies (Sony → JPY, UMC/ASE → TWD) are converted to USD using the **historical exchange rate on each specific reporting date**, not a period average. FX history is fetched via Yahoo Finance (`JPYUSD=X`, `TWDUSD=X`) and the rate closest to each period-end date is used. EUR is handled automatically by Yahoo Finance (FMP returns EUR values already in USD for European companies).

### Output: `sec_financials.csv`

| Column | Description |
|--------|-------------|
| `Company` | Canonical company name |
| `Foreign` | `True` if the company is a foreign filer or foreign-incorporated |
| `Data Source` | `SEC EDGAR`, `FMP`, or `Yahoo Finance` |
| `Form Type` | `10-Q`, `10-K`, `20-F`, `Quarterly`, or `Annual` |
| `Filing date` | Date the filing was submitted (or period end for FMP/Yahoo) |
| `Period end` | End date of the reporting period |
| `R&D Expenses` | USD millions |
| `Revenue` | USD millions |
| `Cost of Revenue` | USD millions |
| `URL` | Link to the SEC filing or Yahoo Finance page |

## Pipeline 2 — Electricity Price Cleaning

Cleans raw EIA (Energy Information Administration) retail electricity price data into a tidy format for analysis.

### How to run

Open `Electricty Price Cleaning/csv_cleaning.ipynb` and run the single cell.

**Input:** `Electricty Price Cleaning/Average_retail_price_of_electricity.csv`  
**Output:** `Electricty Price Cleaning/electricity_prices_tidy.csv`

What it does:
- Reads the wide-format EIA CSV (one row per state × sector, one column per month)
- Filters to **commercial** and **industrial** sectors only
- Melts from wide to long format (one row per state × sector × month)
- Maps full state names to two-letter abbreviations
- Outputs columns: `year`, `month`, `state`, `category`, `price` (cents/kWh)

## Important notes for VS Code users

- **Never save `table_generation.ipynb` from VS Code.** If VS Code prompts "Save changes?" when closing, choose **Don't Save**. If it asks "Overwrite or Revert?", choose **Revert**. The notebook is generated by `write_notebook.py` and VS Code's in-memory version should never be treated as the source of truth.
- **`write_notebook.py` is the source of truth** for all pipeline logic. Edit it, then regenerate.
- **`edgar_cache.json`** can grow large over time. It is safe to delete it — the next run will re-fetch everything from EDGAR and rebuild it.
